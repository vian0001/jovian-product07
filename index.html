1. PENGENALAN ANTARMUKA WINBOX

Dari sumber IDN.ID (2022), Winbox adalah sebuah aplikasi berbasis windows yang biasa digunakan untuk konfigurasi mikrotik. Dari sumber yang sama, setelah berhasil login ke Winbox, kita dapat melihat beberapa tampilan seperti "title bar", Session, Settings, Dashboard, Undo, Redo, dan Safe Mode. Kita juga dapat mengubah identitas router dengan masuk ke menu System â€“ Identity. Dashboard akan menampilkan beberapa menu untuk monitoring seperti CPU Usage, Memory, Time, dan Date.

Dari sumber Biznethome (2025), Winbox adalah aplikasi berbasis Windows yang memungkinkan konfigurasi MikroTik secara grafis melalui GUI (Graphical User Interface). Untuk mengunduh Winbox, pengguna bisa mengakses situs resmi MikroTik: https://mikrotik.com/download dan cukup membuka file Winbox.exe yang telah diunduh tanpa perlu instalasi karena merupakan aplikasi portable.

Dari 2 pendapat tersebut dapat disimpulkan bahwa Winbox adalah aplikasi berbasis Windows yang menyediakan antarmuka grafis (GUI) untuk melakukan konfigurasi MikroTik dengan lebih mudah dan dapat diunduh langsung dari situs resmi MikroTik.

2. MENU UTAMA PADA WINBOX

Dari sumber Biznethome (2025), setelah berhasil login ke Winbox, pengguna akan melihat beberapa menu utama seperti Interfaces yang menampilkan daftar port ethernet yang tersedia, IP yang berisi pengaturan IP Address, DHCP, Firewall, NAT, dan lainnya, serta Bridge yang digunakan untuk menghubungkan beberapa interface agar berfungsi seperti satu jaringan.

Dari sumber IDN.ID (2022), Winbox memiliki menu-menu penting seperti Bridge untuk menggabungkan port di mikrotik menjadi 1 jaringan, Switch untuk konfigurasi VLAN dan Port Isolation, Mesh untuk konfigurasi HWMP+ pada wireless mesh, IP untuk konfigurasi IP Address, DHCP, Firewall, DNS, Hotspot, IPSec, Routes, dan Web Proxy. Menu Routing digunakan untuk konfigurasi Dynamic Routing Protocol seperti RIP, OSPF, BGP. Menu System digunakan untuk Reset Configuration, Packages, Reboot, dan lainnya. Menu Queues digunakan untuk konfigurasi Bandwidth Management.

Dari 2 pendapat tersebut dapat disimpulkan bahwa Winbox menyediakan berbagai menu utama yang memungkinkan pengguna mengakses semua fitur MikroTik seperti pengaturan interface, IP, routing, bandwidth management, dan banyak fitur lainnya melalui antarmuka grafis yang terstruktur.

3. NAVIGASI DASAR PADA WINBOX

Dari sumber Citraweb (tanpa tahun), penggunaan Winbox harian untuk konfigurasi Router Mikrotik sebaiknya menggunakan IP-Address karena akan lebih stabil dengan protocol TCP. Dari sumber yang sama, Mikrotik bisa diremote menggunakan MAC-Address maupun IP-Address. Remote MAC-Address sebaiknya digunakan saat awal melakukan konfigurasi sebelum Router memiliki IP Address saja.

Dari sumber Biznethome (2025), untuk mengakses perangkat MikroTik, pengguna perlu menyambungkan MikroTik ke PC menggunakan kabel LAN, membuka Winbox, dan memilih tab Neighbors untuk mendeteksi perangkat MikroTik yang tersedia. Kemudian pilih perangkat dengan MAC Address yang terdeteksi, masukkan Username (default: admin) dan kosongkan Password (default tidak ada password), dan klik Connect untuk masuk ke antarmuka konfigurasi MikroTik.

Dari 2 pendapat tersebut dapat disimpulkan bahwa untuk navigasi dasar pada Winbox, pengguna dapat mengakses router MikroTik melalui MAC Address (saat awal konfigurasi) atau IP Address (untuk penggunaan harian yang lebih stabil), dengan proses koneksi yang sederhana melalui tab Neighbors di Winbox.

4. FITUR-FITUR PENTING PADA WINBOX

Dari sumber Citraweb (tanpa tahun), untuk meminimalkan kesalahan konfigurasi, pengguna dapat menggunakan tombol "Safe Mode". Pada saat Safe mode diaktifkan, konfigurasi yang dilakukan tidak akan langsung diterapkan secara permanen. Dari sumber yang sama, Dashboard winbox bisa digunakan untuk menampilkan beberapa informasi yaitu Time, Date, CPU load, memory, dan Uptime. Informasi yang ditampilkan merupakan informasi real time. CPU menunjukkan penggunaan CPU Router, Memory menunjukkan sisa RAM yang ada di router, dan Uptime menunjukkan lama waktu Router sudah aktif. Dengan adanya informasi yang tertampil langsung di dashboard akan lebih memudahkan monitoring Router saat melakukan remote winbox.

Dari sumber Citraweb (tanpa tahun), untuk mengupload maupun download file ke router, dengan Winbox versi baru sudah tersedia tombol upload jika ingin menambahkan file ke dalam router. Untuk mengambil file dari Router, bisa dilakukan dengan memilih file kemudian klik kanan download. Dari sumber yang sama, Winbox Loader juga dapat melakukan penyimpanan data remote Router baik untuk IP Address, Username maupun Password dengan menekan tombol Add/Set sebelum klik tombol Connect. Informasi Router akan tersimpan pada tab Managed dan untuk melakukan remote ulang, tinggal double click pada list Router yang diinginkan. Untuk memindahkan data ke laptop/PC lain, pengguna bisa melakukan export import untuk list yang sudah ada di winbox loader sebelumnya.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa Winbox menyediakan fitur-fitur penting seperti Safe Mode untuk mencegah kesalahan konfigurasi, Dashboard untuk monitoring real-time, fitur upload/download file, dan kemampuan menyimpan informasi router untuk memudahkan koneksi di masa mendatang.

5. PENGENALAN COMMAND LINE INTERFACE (CLI) MIKROTIK

Dari sumber Nevacloud (2024), CLI (Command Line Interface) MikroTik adalah sebuah antarmuka berbasis teks yang digunakan untuk mengonfigurasi perangkat MikroTik, termasuk router dan switch. Melalui CLI, pengguna dapat menjalankan perintah-perintah khusus untuk mengatur dan memodifikasi konfigurasi perangkat secara manual, yang berguna untuk pengaturan dengan tingkat detail dan kustomisasi tinggi. CLI menyediakan kontrol lebih detail atas konfigurasi perangkat dan memungkinkan automatisasi tugas-tugas rutin melalui skrip untuk menghemat waktu dan mengurangi kemungkinan kesalahan manusia.

Dari sumber PCNetLabs (2020), untuk mengkonfigurasi Mikrotik Router ada beberapa cara termasuk melalui perintah CLI atau Command Line Interface. Mengkonfigurasi Mikrotik dengan mengetikkan perintah CLI memang lebih menantang daripada lewat Winbox atau browser karena minimal pengguna harus mengetahui logika atau susunan baris perintahnya. Namun, mengkonfigurasi Mikrotik menggunakan CLI mempunyai beberapa kelebihan, seperti kemampuan untuk mengakses Mikrotik melalui telnet, ssh, kabel serial, menu terminal pada winbox, atau secara langsung.

Dari 2 pendapat tersebut dapat disimpulkan bahwa CLI MikroTik adalah antarmuka berbasis teks yang menawarkan kontrol lebih detail dan kemampuan otomatisasi, meskipun membutuhkan pemahaman logika perintah yang lebih dalam dibandingkan dengan menggunakan Winbox.

Mikrotik mempunyai fitur Command Line Completion sehingga pengguna cukup mengetik 3-5 huruf lalu tekan tab, misalnya untuk mengetikkan "interfaces" cukup ketik "int" lalu tab. Penggunaan tab dan "?" juga bisa digunakan untuk melihat perintah selanjutnya setelah mengetikan sebuah perintah.

Dari sumber Blues Pedia (tanpa tahun), pada dasarnya mikrotik adalah router berbasis Linux sehingga tampilannya hitam-putih dengan mode teks. Untuk melihat induk perintah mikrotik tekan saja tab atau tanda tanya "?". Fitur Command Line Completion memungkinkan pengguna cukup mengetik 3-5 huruf lalu tekan tab untuk melengkapi perintah. Penggunaan tab dan "?" juga bisa digunakan untuk melihat perintah selanjutnya setelah mengetikan sebuah perintah.

Dari 2 pendapat tersebut dapat disimpulkan bahwa navigasi dasar pada CLI MikroTik memanfaatkan fitur Command Line Completion dengan tombol Tab dan tanda tanya "?" untuk memudahkan pengguna dalam mengetikkan dan melihat perintah-perintah yang tersedia.

7. PERINTAH DASAR CLI MIKROTIK

Dari sumber Nevacloud (2024), beberapa perintah dasar yang sering digunakan dalam CLI MikroTik dan fungsinya adalah "/interface print" untuk menampilkan daftar semua interface dan "/ip address add" untuk menambahkan alamat IP ke sebuah interface.

Dari sumber Blues Pedia (tanpa tahun), beberapa perintah dasar CLI MikroTik yang sering digunakan antara lain: 1) "add" untuk menambahkan properties (contoh: # ip address add address=192.168.1.1/24 interfaces=ether1), 2) "comment" untuk memberi komentar sebagai keterangan, 3) "disable" untuk menonaktifkan service (contoh: # ip address disable 0), 4) "enable" untuk mengaktifkan service (contoh: # ip address enable 0), 5) "edit" untuk mengubah/edit sebuah file/properties (contoh: # ip address edit 0 comment).

Dari sumber Komputer Jaringan (tanpa tahun), beberapa perintah dasar mikrotik lainnya meliputi: 1) Perintah untuk mengganti hostname pada Router MikroTik, 2) Perintah untuk melihat identitas yang sudah dikonfigurasi, 3) Perintah untuk mematikan Router MikroTik, 4) Perintah untuk merestart Router MikroTik, 5) Perintah untuk melakukan reset konfigurasi pada router, 6) Perintah untuk menyimpan data agar jika konfigurasi terhapus dapat mengambil data yang telah dibackup/disimpan sebelumnya, 7) Perintah untuk melihat paket yang telah terinstall pada Router Mikrotik.

Dari 3 pendapat tersebut dapat disimpulkan bahwa CLI MikroTik memiliki berbagai perintah dasar yang memungkinkan pengguna untuk mengkonfigurasi dan mengelola router, seperti menampilkan interface, menambahkan alamat IP, mengaktifkan/menonaktifkan service, mengedit properties, mengganti hostname, merestart router, dan melakukan backup konfigurasi.

8. KONFIGURASI IP ADDRESS MELALUI CLI

Terminal menampilkan perintah `ip address add` untuk menetapkan alamat IP ke antarmuka jaringan. Sumber: Jovian

Dari sumber Herza Cloud (2020), untuk menginput IP pada MikroTik melalui CLI, pengguna dapat menggunakan perintah "/ip address add address=192.168.1.10/24 interface=ether1" untuk menambahkan alamat IP dan "/ip route add gateway=192.168.1.1" untuk menambahkan gateway.

Dari sumber Nevacloud (2024), salah satu langkah pertama dalam konfigurasi jaringan adalah menetapkan alamat IP untuk interface pada router dengan perintah "/ip address add address=192.168.88.1/24 interface=ether1". Ini menetapkan alamat IP 192.168.88.1 dengan subnet mask 255.255.255.0 ke interface ether1.

Dari 2 pendapat tersebut dapat disimpulkan bahwa konfigurasi IP address melalui CLI MikroTik dilakukan dengan perintah "/ip address add" yang memungkinkan pengguna untuk menentukan alamat IP, subnet mask, dan interface yang akan digunakan, serta "/ip route add" untuk mengatur gateway.

/ip address add address=192.168.1.10/24 interface=ether1 /ip route add gateway=192.168.1.1

9. KONFIGURASI ROUTING MELALUI CLI

Terminal menunjukkan penambahan rute gateway menggunakan perintah `ip route add` untuk mengarahkan lalu lintas jaringan. Sumber: Jovian

Dari sumber Nevacloud (2024), routing adalah proses mengatur lalu lintas jaringan ke tujuan yang benar. Untuk mengatur routing dasar pada MikroTik, pengguna dapat menambahkan route statis dengan perintah "/ip route add dst-address=0.0.0.0/0 gateway=192.168.88.1". Ini adalah contoh menambahkan default gateway, yang akan digunakan untuk mengarahkan lalu lintas ke internet atau jaringan lain.

Dari sumber Herza Cloud (2020), setelah menambahkan alamat IP ke interface, pengguna perlu menambahkan route agar dapat terhubung ke internet dengan perintah "/ip route add gateway=192.168.1.1", lalu memeriksa konfigurasi dengan "/ip route print".

Dari 2 pendapat tersebut dapat disimpulkan bahwa konfigurasi routing melalui CLI MikroTik dilakukan dengan perintah "/ip route add" untuk menambahkan route statis atau default gateway yang akan mengarahkan lalu lintas jaringan ke tujuan yang benar, dan dapat diperiksa dengan perintah "/ip route print".

/ip route add dst-address=0.0.0.0/0 gateway=192.168.88.1 /ip route print

10. KONFIGURASI FIREWALL MELALUI CLI

Terminal menampilkan perintah `ip firewall filter add` untuk membuat aturan baru dalam konfigurasi firewall. Sumber: Jovian

Dari sumber Nevacloud (2024), konfigurasi firewall melalui CLI MikroTik melibatkan pembuatan aturan (rules) yang menentukan jenis trafik yang diizinkan atau diblokir. Contoh konfigurasi dasar firewall adalah membuat aturan untuk memblokir trafik dari alamat IP tertentu dengan perintah "/ip firewall filter add chain=input src-address=192.168.88.100 action=drop". Ini akan memblokir semua lalu lintas masuk dari IP 192.168.88.100.

Dari sumber Labkom.co.id (tanpa tahun), perintah-perintah CLI MikroTik dapat digunakan untuk berbagai konfigurasi termasuk firewall. Mikrotik memiliki fitur Command Line Completion yang memudahkan pengguna dalam mengetikkan perintah-perintah firewall dan konfigurasi lainnya.

Dari 2 pendapat tersebut dapat disimpulkan bahwa konfigurasi firewall melalui CLI MikroTik memungkinkan pengguna untuk membuat aturan-aturan yang mengontrol lalu lintas jaringan, seperti memblokir trafik dari alamat IP tertentu, dengan memanfaatkan struktur perintah yang logis dan fitur Command Line Completion untuk memudahkan pengetikan.

/ip firewall filter add chain=input src-address=192.168.88.100 action=drop

11. PEMANTAUAN JARINGAN MELALUI CLI

Terminal menjalankan perintah `monitor-traffic` untuk melihat statistik lalu lintas data secara langsung pada antarmuka jaringan. Sumber: Jovian

Dari sumber Nevacloud (2024), mengawasi lalu lintas jaringan adalah aspek penting dari manajemen jaringan. MikroTik CLI menyediakan berbagai perintah untuk memonitor aktivitas jaringan secara real-time, seperti "/interface monitor-traffic" yang menampilkan statistik trafik untuk interface tertentu dan dapat digunakan untuk memantau jumlah data yang masuk dan keluar dari sebuah interface. Ada juga "/tool torch" yang memberikan tampilan lebih rinci tentang paket yang melewati interface, termasuk alamat IP sumber dan tujuan, serta port.

Dari sumber Biznethome (2025), untuk mengecek koneksi ke internet melalui CLI, pengguna dapat memasukkan perintah "ping 8.8.8.8" di New Terminal. Jika berhasil, akan muncul balasan dari server Google. Pengguna juga dapat masuk ke Log untuk melihat riwayat aktivitas dan error, dan memastikan tidak ada error terkait DHCP, NAT, atau koneksi internet.

Dari 2 pendapat tersebut dapat disimpulkan bahwa CLI MikroTik menyediakan berbagai perintah untuk pemantauan jaringan, seperti monitor-traffic untuk statistik trafik, torch untuk analisis paket yang lebih detail, ping untuk mengecek konektivitas, dan log untuk melihat riwayat aktivitas dan error.

/interface monitor-traffic ether1 ping 8.8.8.8

12. TIPS DAN TRIK MENGGUNAKAN CLI MIKROTIK

Dari sumber Nevacloud (2024), menggunakan CLI secara efisien melibatkan penguasaan pintasan keyboard dan alih aksen. Beberapa tips penggunaan CLI mencakup menggunakan tombol Tab untuk autocompletion, yang akan mengisi otomatis bagian dari perintah yang sedang diketik, menghemat waktu dan mengurangi kesalahan.

Dari sumber Citraweb (tanpa tahun), MikroTik menyediakan menu Setup pada Command Line Interface yang dapat digunakan untuk melakukan setting dasar. Untuk mengakses menu ini, bisa dilakukan dengan mengetikkan command "Setup". Ada beberapa konfigurasi dasar yang dapat dilakukan pada menu Setup, seperti DHCP Server & Client, IP Address, Default Gateway, PPPoE Client, PPTP Client, serta reset semua konfigurasi router. Pada saat perintah setup dipanggil, Router berada dalam Safe Mode, artinya konfigurasi yang dilakukan hanya akan disimpan sementara. Jika konfigurasi tersebut menyebabkan Router error, maka safe mode akan menghapus konfigurasi yang dibuat dan mengembalikan Router pada konfigurasi sebelumnya. Untuk menyimpan setting, bisa menggunakan tombol CTRL+X untuk keluar dari menu setup sekaligus melepas (release) Safe Mode Router.

Dari 2 pendapat tersebut dapat disimpulkan bahwa penggunaan CLI MikroTik dapat dioptimalkan dengan memanfaatkan fitur autocompletion dengan tombol Tab, menggunakan menu Setup untuk konfigurasi dasar, dan memahami fungsi Safe Mode serta penggunaan tombol CTRL+X untuk menyimpan konfigurasi secara permanen.

13. KODE CLI PENTING PADA MIKROTIK

Antarmuka baris perintah MikroTik menampilkan daftar antarmuka jaringan menggunakan perintah `interface print`. Sumber: Jovian

Dari sumber Nevacloud (2024), beberapa kode CLI penting yang sering digunakan dalam konfigurasi MikroTik adalah:

/interface print

Perintah ini digunakan untuk menampilkan semua interface yang ada pada router MikroTik beserta statusnya. Dari sumber yang sama, perintah ini sangat berguna untuk identifikasi awal interface sebelum melakukan konfigurasi lebih lanjut.

Dari sumber Tech@Layer-x.com (2025), kode CLI penting berikutnya adalah untuk konfigurasi NAT yang memungkinkan jaringan lokal mengakses internet:

/ip firewall nat add chain=srcnat action=masquerade out-interface=ether1

Perintah ini akan membuat aturan NAT yang mengizinkan paket dari jaringan lokal untuk keluar melalui interface ether1 (biasanya interface WAN) dengan mengubah alamat sumber menjadi alamat IP publik router.

Dari sumber PCNetLabs (2020), untuk mengamankan akses ke router, dapat digunakan perintah berikut untuk mengubah password administrator:

/user set admin password=password_baru

Perintah ini mengubah password untuk pengguna admin yang merupakan langkah dasar untuk keamanan router.

Dari 3 pendapat tersebut dapat disimpulkan bahwa kode-kode CLI penting pada MikroTik mencakup perintah untuk menampilkan interface, konfigurasi NAT untuk akses internet, dan pengubahan password administrator untuk keamanan dasar.

14. KODE CLI UNTUK KONFIGURASI DHCP SERVER

Dari sumber MikroTik Documentation (2025), konfigurasi DHCP server dapat dilakukan dengan perintah berikut:

/ip dhcp-server setup

Perintah ini akan menjalankan wizard setup DHCP server dengan cara interaktif, dimana pengguna akan diminta untuk memasukkan parameter-parameter seperti interface, network address, gateway, pool address, dan DNS server. Dari sumber yang sama, cara ini merupakan cara termudah untuk menyiapkan DHCP server pada router MikroTik.

Dari sumber Biznethome (2025), untuk konfigurasi DHCP server secara manual, dapat menggunakan serangkaian perintah berikut:

/ip pool add name=dhcp_pool1 ranges=192.168.88.2-192.168.88.254 /ip dhcp-server add address-pool=dhcp_pool1 interface=ether2 name=dhcp1 /ip dhcp-server network add address=192.168.88.0/24 gateway=192.168.88.1 dns-server=8.8.8.8,8.8.4.4

Perintah-perintah di atas akan membuat IP pool untuk DHCP, menambahkan DHCP server pada interface ether2, dan mengkonfigurasi network parameter termasuk gateway dan DNS server.

Dari 2 pendapat tersebut dapat disimpulkan bahwa konfigurasi DHCP server pada MikroTik dapat dilakukan baik dengan menggunakan wizard setup yang interaktif maupun dengan serangkaian perintah manual yang memungkinkan kontrol lebih detail atas parameter konfigurasi.

15. KODE CLI UNTUK KONFIGURASI FIREWALL

Terminal menampilkan beberapa aturan firewall yang telah dikonfigurasi menggunakan perintah `ip firewall filter add`. Sumber: Jovian

Dari sumber Nevacloud (2024), berikut adalah contoh kode CLI untuk konfigurasi firewall dasar pada MikroTik:

/ip firewall filter add chain=input protocol=tcp dst-port=22 action=accept comment="Allow SSH" /ip firewall filter add chain=input connection-state=established,related action=accept comment="Allow Established Connections" /ip firewall filter add chain=input connection-state=invalid action=drop comment="Drop Invalid Connections" /ip firewall filter add chain=forward connection-state=established,related action=accept comment="Allow Established Connections" /ip firewall filter add chain=forward connection-state=invalid action=drop comment="Drop Invalid Connections" /ip firewall filter add chain=input action=drop comment="Drop everything else"

Dari sumber yang sama, rangkaian perintah ini membuat aturan firewall dasar yang mengizinkan koneksi SSH, menerima koneksi yang sudah dibuat sebelumnya, menolak koneksi yang tidak valid, dan menolak semua koneksi masuk lainnya.

Dari sumber Tech@Layer-x.com (2025), untuk membatasi akses pada port tertentu, dapat digunakan perintah firewall berikut:

/ip firewall filter add chain=input protocol=tcp dst-port=80,443 src-address=192.168.1.0/24 action=accept comment="Allow LAN Web Access"

Perintah ini mengizinkan akses HTTP dan HTTPS hanya dari jaringan 192.168.1.0/24 ke router.

Dari 2 pendapat tersebut dapat disimpulkan bahwa konfigurasi firewall pada MikroTik melalui CLI memungkinkan pengguna untuk membuat aturan keamanan yang komprehensif untuk mengontrol lalu lintas jaringan, baik untuk melindungi router itu sendiri maupun untuk mengatur akses antar jaringan.

16. KODE CLI UNTUK MONITORING DAN TROUBLESHOOTING

Perintah `monitor-traffic` menampilkan statistik lalu lintas data secara real-time pada antarmuka jaringan. Sumber: Jovian
Perintah `ping` digunakan untuk menguji konektivitas jaringan ke server Google. Sumber: Jovian

Dari sumber Nevacloud (2024), beberapa perintah CLI penting untuk monitoring dan troubleshooting jaringan pada MikroTik adalah:

/interface monitor-traffic ether1

Perintah ini akan menampilkan statistik traffic secara real-time pada interface ether1, menunjukkan jumlah data yang masuk dan keluar dalam bentuk bit per detik.

Dari sumber MikroTik Documentation (2025), untuk melakukan analisis traffic yang lebih detail, dapat digunakan perintah:

/tool torch interface=ether1 ip-protocol=tcp port=80 src-address=192.168.1.0/24

Perintah ini akan melakukan monitoring packet dengan detail paket yang melewati interface ether1 dengan protokol TCP pada port 80 dari sumber alamat 192.168.1.0/24.

Dari sumber Citraweb (tanpa tahun), untuk backup konfigurasi router yang merupakan bagian penting dari manajemen dan troubleshooting, dapat digunakan perintah:

/system backup save name=backup-config

Perintah ini akan membuat file backup dari konfigurasi router saat ini dengan nama backup-config.

Dari 3 pendapat tersebut dapat disimpulkan bahwa CLI MikroTik menyediakan berbagai perintah untuk monitoring dan troubleshooting jaringan, dari pemantauan traffic sederhana hingga analisis packet yang mendalam, serta kemampuan untuk membuat backup konfigurasi untuk pemulihan jika terjadi masalah.
